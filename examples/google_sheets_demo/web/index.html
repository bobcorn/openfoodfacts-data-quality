Current demo limits
  • The UI loads CSV files into the Data tab and does not support JSONL.
  • Validation runs the global checks used in this demo. It does not include Canada rules.
  • Findings are shown per row, not at the exact cell level.
  • Prepare upload candidates only keeps rows whose dq_status is ok and writes them into the output tab without the derived validation columns.
  • Upload to Open Food Facts is not implemented yet. The button only shows the planned next step.
  • Gemini is not controlled by this app. If it is available in Google Sheets, you can use it in the side panel while this demo runs validation.
  • The app reads and writes the full tab for its main actions. Large sheets will feel slower than the 1,000 row demo file.
How this demo uses your data

This demo runs in Docker on your machine. Google login and sheet access stay in the browser and are limited to the spreadsheet you choose.

When you upload a CSV, the file content goes to the local demo server on localhost so it can parse the table. When you run validation, the current sheet rows go to that same local server so the Python library can run the checks. The Google access token is used only for browser requests to Google and is not sent to the local Python server.

Steps that only reshape sheet data, such as clearing dq_* columns or preparing upload candidates, run in the browser before the updated data is written back to Google Sheets. The selected spreadsheet ID is stored in the browser so the page can remember it. The Google access token is kept only for the current browser session.

This demo does not send sheet rows to Open Food Facts because the upload button is still a placeholder. Outside Google services, the only backend is the local container on your machine.
